PROJECT: DekoKraft (Static site on GitHub Pages)
STACK: HTML + CSS + Vanilla JavaScript + JSON فقط (بدون Framework, بدون Backend)

GOAL:
موقع ثابت بتصميم Dark/Glass. يدعم AR و DE. عرض أقسام + منتجات مختارة + صفحة عرض منتج تفاعلية + Kontakt موحد.

PAGES:
1) index.html
- Hero
- Sections Grid (4 بطاقات أقسام رئيسية)
  - كل بطاقة قسم رئيسية = “Main Section Card”
  - تحتوي: صورة ممثلة (أول منتج في القسم) + زر "عرض القسم"
  - IMPORTANT: الضغط على زر "عرض القسم" يأخذ إلى product.html?id=... (صفحة عرض منتج)
  - البطاقة أيضاً يمكن جعلها قابلة للنقر (اختياري) مع الحفاظ على زر
- Featured Products Grid (صور عشوائية/منتجات مختارة)
  - “Random/Featured Product Cards”
  - لا تحتوي زر
  - الصورة/البطاقة نفسها تكون رابط إلى product.html?id=...
  - يجب عدم تكرار نفس المنتج المستخدم كصورة ممثلة للقسم ضمن المنتجات المختارة
- Kontakt

2) product.html (صفحة العرض المطلوبة)
- عند فتح المنتج:
  - صورة كبيرة (Main Viewer)
  - عنوان + وصف قصير وجذاب
- ثم أسفل/جانب الصورة:
  - قائمة عمودية (على الكمبيوتر) / أفقية (على الهاتف) لمنتجات نفس القسم (thumbnails)
  - كل thumbnail عند الضغط عليها:
    - تستبدل الصورة الكبيرة + العنوان + الوصف لهذا المنتج فوراً (بدون فتح صفحة جديدة)
    - ويمكن تحديث الرابط عبر history.pushState (?id=...) حتى يبقى الرابط صحيحاً
- Kontakt

3) category.html (اختياري)
- يبقى موجوداً لكنه غير أساسي في الـ UX الحالي
- يمكن استخدامه لاحقاً كصفحة عرض كل منتجات القسم
- حالياً الروابط الرئيسية من “عرض القسم” تتجه إلى product.html كما هو مطلوب

ASSETS:
- assets/css/main.css: كل التنسيقات (Glass/Dark + Responsive + Grid + Cards + Kontakt + Product viewer)
- assets/js/main.js: المنطق (Menu + AR/DE + تحميل JSON + Rendering + viewer thumbnails)
- assets/data/products.json: بيانات المنتجات
- assets/images/icons: whatsapp.svg, telegram.svg, email.svg (تم حذف facebook نهائياً)
- assets/images/logo: logo-dekokraft-600.webp + logo-dekokraft-1200.webp
- assets/images/products/*: صور المنتجات

LANGUAGES:
- AR = RTL, DE = LTR
- حفظ اللغة في localStorage
- تبديل اللغة لا يغيّر الروابط ولا يكسر التصميم

CRITICAL IMAGE RULE:
في products.json نستخدم:
imageBase: "assets/images/products/<cat>/<name>"
والكود يختار:
- يفضّل -1200.webp إن كان موجوداً
- وإلا -600.webp إن كان موجوداً
- وإلا placeholder

VISUAL DISTINCTION RULE:
- Main Section Cards: شكل نظيف وصورة واضحة (بدون “بخار/زجاج” فوق الصورة)
- Featured/Random Product Cards: تأثير “ضباب على الزجاج” (Decor overlay) يظهر فقط عليها، لإبراز الفرق

DESKTOP LAYOUT RULE:
- تجنّب جعل البطاقات صغيرة جداً على الكمبيوتر
- عدد الأعمدة للمنتجات يكون مرناً لكن لا يصبح كثيفاً (حد أقصى 3 أعمدة للمنتجات على شاشات عادية)
- الأقسام: 4 أعمدة على الكمبيوتر، و 2×2 على الهاتف

WORKFLOW:
عند كل تحديث: طباعة PROJECT_PROMPT.txt و PROJECT_STATE.txt ثم طباعة الأكواد كاملة.
