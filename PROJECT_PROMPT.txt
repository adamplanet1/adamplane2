PROJECT: DekoKraft (Static site on GitHub Pages)
REPO: adamplane2
STACK: HTML + CSS + Vanilla JavaScript + JSON فقط
NO Frameworks, NO Backend, NO Build tools

GOAL
إنشاء موقع ثابت أنيق (Dark/Glass) لمشروع DekoKraft.
يدعم لغتين: العربية (AR) + الألمانية (DE).
التبديل بين اللغات لا يغير الروابط ولا يكسر التصميم.
AR = RTL ، DE = LTR.

PAGES
1) index.html
- Hero
- قسم "الأقسام الرئيسية" (4 بطاقات): Gifts / Decor / Kids / Services
  REQUIRED:
  - البطاقات الرئيسية يجب تكون 2×2 على الهاتف
  - وتبقى 2×2 على الكمبيوتر أيضاً (لا تتحول إلى 4×4)
  - كل بطاقة قسم تحتوي:
    (1) صورة تمثل القسم (هي أول منتج في هذا القسم)
    (2) عنوان + وصف
    (3) زر واضح: "عرض المجموعة"
  NAVIGATION:
  - زر "عرض المجموعة" يفتح product.html?id=FIRST_PRODUCT_ID مباشرة
    (لا يذهب إلى category.html)

- قسم "منتجات مختارة" (Featured / Random)
  REQUIRED:
  - هذه البطاقات بدون زر داخلها
  - البطاقة كلها تكون رابط إلى product.html?id=...
  - يجب عدم تكرار منتجات الأقسام الرئيسية
    (استبعاد firstProductId لكل قسم)
  VISUAL:
  - الأقسام الرئيسية: Glass نظيف بدون بخار أزرق
  - المنتجات المختارة: يسمح بديكور (Blue Fog / Glass Vapor) خفيف وواضح
    لكن لا يظهر كأنه صورة مكسورة.

- KONTAKT (موحد في كل الصفحات)
  REQUIRED DESIGN:
  - لا نريد مربع/خلفية منفصلة لمحتوى KONTAKT (يندمج مع خلفية الصفحة)
  - يظهر Logo ثم تحته مباشرة أيقونات التواصل (بدون خلفيات زر)
  - ثم QR-Code لصورة الموقع
  CONTENT:
  - WhatsApp + Telegram + Email فقط
  - ممنوع Facebook (لا كود ولا SVG ولا روابط)

2) product.html
- صفحة عرض منتج واحد:
  REQUIRED:
  - صورة كبيرة رئيسية
  - عنوان + وصف قصير جميل
  - عند الضغط على الصورة الكبيرة: تفتح الصورة الأصلية (fullscreen / overlay)
    بدون عنوان وبدون وصف.
- Slider / Gallery للمنتج:
  REQUIRED:
  - المنتج قد يملك 6 صور زوايا (gallery)
  - تظهر كشريط تمرير (slider) أو thumbnails قابلة للتمرير
  - عند اختيار صورة من الـ gallery:
    - تتغير الصورة الكبيرة
- منتجات نفس القسم (Similar / Same category):
  REQUIRED:
  - قائمة/شبكة لمنتجات من نفس القسم
  - عند الضغط على منتج مشابه:
    - يبقى في product.html
    - يتغير المحتوى (الصورة/العنوان/الوصف/gallery)
    - ويتغير رابط الصفحة باستخدام history.replaceState إلى id الجديد
    - وكأنه فتحت صفحة جديدة بدون reload
- KONTAKT في الأسفل

3) category.html (اختياري)
- Grid لمنتجات قسم حسب ?cat=
- كل بطاقة تفتح product.html?id=...
- KONTAKT في الأسفل
NOTE:
- الصفحة الرئيسية لا تعتمد عليها للتنقل الرئيسي (الأزرار تذهب مباشرة للمنتج الأول)

DATA (products.json)
- قاعدة البيانات تشمل:
  id, category, title_ar, title_de, desc_ar, desc_de, imageBase, galleryCount
- CRITICAL IMAGE RULE:
  الصور ليست باسم gift-001.webp
  بل:
    gift-001-1200.webp و/أو gift-001-600.webp
  و للـ Gallery:
    gift-001-1-1200.webp ... gift-001-6-1200.webp (إن وُجد)
- imageBase مثال:
  "assets/images/products/gifts/gift-001"
- اختيار الصورة:
  - يفضّل -1200.webp إن كان موجوداً
  - وإلا -600.webp إن كان موجوداً
  - وإلا fallback بدون كسر التصميم

WORKFLOW RULES (IMPORTANT)
1) خطوة بخطوة
2) عند أي تعديل في كود: اطبع الملف كاملاً (بدون قص ولصق أجزاء)
3) ممنوع إضافة Facebook أو أي ملف غير موجود في الشجرة
4) حافظ على استقرار التصميم: Responsive + No overflow + Menu يعمل على الهاتف
